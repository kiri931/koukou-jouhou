1. なぜ使われる？（目的）

理解の分割（層で整理） 相互運用（異メーカー間の互換性） トラブル切り分け（層で原因特定）
  • 通信の仕組みを分けて考える：物理（ケーブル）とアプリ（HTTP等）を切り離して理解。
  • 標準化：同じルールで作れば、機器・OSが違ってもつながる。
  • 障害対応：どの層の故障か（配線？IP？TCP？）を素早く特定。

OSIは理論モデル、TCP/IPは実運用の事実上標準という立ち位置です。

4. 各層の代表的プロトコル（TCP/IPモデル基準）

TCP/IP層 対応するOSI層 主な役割 代表的プロトコル・例
アプリケーション層 OSIの7〜5層 アプリ同士のやり取り HTTP/HTTPS（Web）、FTP（ファイル転送）、SMTP／POP3／IMAP（メール）、DNS、SSH、TLS など
トランスポート層 OSIの4層 信頼性・順序制御・多重化 TCP（信頼性あり）、UDP（軽量・高速）
インターネット層 OSIの3層 ルーティング・到達性 IP（IPv4/IPv6）、ICMP（通知/診断）、ARP（アドレス解決）
ネットワークインターフェース層 OSIの2〜1層 物理/リンク伝送 Ethernet、Wi-Fi、PPP、MACアドレス、フレーム、電気信号

※ 「DNSはアプリ層」「ARPはインターネット層寄り（L3補助）」のように、試験では位置づけを問われがちです。
💡 例：Pingはインターネット層(ICMP)を利用して疎通確認を行い、Tracerouteは経路上のルータを確認するツールです。

5. 通信の流れ（カプセル化のイメージ）

送信側（アプリ → 下位層へ）
▶ ネットワーク ▶
受信側（上位層へ）
[アプリデータ]
↓ ヘッダ付与（アプリ層 / TLSなど）
[App H][データ]
↓ セグメント化（トランスポート層: TCP/UDP）
[TCP/UDP H][App H][データ]
↓ ルーティング情報付与（インターネット層: IP）
[IP H][TCP/UDP H][App H][データ]
↓ フレーム化（ネットワークIF層: Ethernet / Wi-Fi）
[MAC H][IP H][TCP/UDP H][App H][データ][FCS]

受信側ではこの順でヘッダをはがす（逆カプセル化）ことで元のデータに戻します。

6. 背景と成り立ち

  • OSIモデルは ISO（国際標準化機構） により提案された理論モデル（1980年代）。
  • 一方、TCP/IP はアメリカのARPANETをもとに実用化され、インターネットの標準へ。
  • 現在は「概念理解にOSI」「実際の通信ではTCP/IP」という使い分けが一般的です。

OSIは学習・教育用の枠組み、TCP/IPは現実の通信の基盤。

9. まとめ（試験・実務の要点）

  • OSIは学習用の7層、TCP/IPは実運用の4層（対応関係を覚える）
  • 障害対応は層で切り分け：配線（1–2層）、IP設定（3層）、TCP/UDP（4層）、アプリ設定（上位層）
  • 代表プロトコルの所在：HTTPはアプリ層 / TCPはトランスポート / IPはインターネット / EthernetはネットワークIF
